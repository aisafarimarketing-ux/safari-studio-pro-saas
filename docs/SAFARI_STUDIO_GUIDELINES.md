# CORE ARCHITECTURAL INVARIANTS

## PRINT DISCIPLINE

Print PDF is the ONLY export source.

All export paths must flow through:
PrintProposalDocument
→ composer
→ PdfFit pages.

Web View and Spread View must NEVER export directly.

---

## PAGE OWNERSHIP

ONE DAY = ONE PAGE.
ONE PROPERTY = ONE PAGE.

Editorial pages are:

* fixed compositions
* deterministic
* non-paginating
* truncation-based

No continuation pages.
No spill-based rendering.

Content adapts to layout.
Layout does NOT endlessly adapt to content.

---

## PDF SYSTEM DISCIPLINE

PdfFit pages must remain:

* mm-coordinate based
* slot-based
* deterministic
* manifest-driven

Never introduce:

* uncontrolled flow rendering
* flex/grid composition inside pdf-page
* browser screenshot PDF logic
* uncontrolled auto-pagination

Every text slot must define:

* max_chars
* overflow_behavior

---

## COMPOSER GOVERNANCE

composePdfPages must remain:

* pure
* deterministic
* side-effect free
* independent of React/DOM concerns

The composer architecture is considered foundational.

Adjacency-based composition is intentional and must remain.

The system composes:
story units,
not arbitrary pages.
